Github+hexo博客搭建

发表于 2018-03-28 | 分类于 杂文

博客记录个人信息，一般可以使用现成的各大门户网站提供的博客系统，然后自己把信息记录上去，然后作为一个程序员，我觉得还是自己搭建一个博客系统显得那么高大上。下面是就采用Github和Hexo进行搭建。

安装Hexo

安装node.js

到node.js官网下载node.js，并且安装

安装Hexo

在本地磁盘中新建一个目录比如D:\hexo，用于安装Hexo。
1.进入D:\hexo文件夹中
2.在命令行中执行:
$ npm install -g hexo-cli
3.Hexo安装完成后，执行下面命令，该命令用于下载相关web页面
$ hexo init
$ cd
$ npm install
注：表示要创建的博客的文件夹名称
4.创建好博客的目录结构内容

创建Github博客仓库

编写博客与GitHub部署

新建博客

1.在用户自定义的博客目录下执行如下命令：
$ hexo new [layout] <title>

新建一个layout型的文件<title>表示文件类型，[layout]可以省略，如果省略表示创建一篇普通博客。
[layout]通常有多种类型：post（文章详情页）、page（导航标签页）、draft

2.博客中插入资源(下面内容引用自Hexo官网)
对于那些想要更有规律地提供图片和其他资源以及想要将他们的资源分布在各个文章上的人来说，Hexo也提供了更组织化的方式来管理资源。这个稍微有些复杂但是管理资源非常方便的功能可以通过将 config.yml 文件中的 post_asset_folder 选项设为 true 来打开。
_config.yml
post_asset_folder: true
当资源文件管理功能打开后，Hexo将会在你每一次通过 hexo new [layout] <title> 命令创建新文章时自动创建一个文件夹。这个资源文件夹将会有与这个 markdown 文件一样的名字。将所有与你的文章有关的资源放在这个关联文件夹中之后，你可以通过相对路径来引用它们，这样你就得到了一个更简单而且方便得多的工作流。
比如：
{ % asset_path slug %}
{ % asset_img slug [title] %}
{ % asset_link slug [title] %}
下面的图片的引入方式：
{ % asset_img example.jpg This is an example image %}

3.创建分类：
hexo new page <title>
title 是自定义的类型:

部署

新建好博客后，执行下面命令
文章编辑好之后，运行生成、部署命令：
hexo clean //清理
hexo g //生成静态文件
hexo d //部署
当然你也可以执行下面的命令，相当于上面两条命令的效果
hexo clean
hexo d -g
